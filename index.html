工作表结构
  • 仓位看板：汇总各标的当前先验/后验概率、建议凯利仓位，自动从各标的记录表最新一行取数，无需手动同步。
  • 贝叶斯更新/凯利仓位记录表：周大生、泸州老窖、五粮液、平安银行四只标的的贝叶斯更新记录表，每次财报或重大事件发布后新增一行。
  • 凯利速算器：独立的 what-if 速算区，输入概率和赔率即可即时看仓位建议，不影响正式记录。
每次更新时怎么填（以记录表的一行为例）
  1. 日期、触发事件：填写财报/公告发布日期与关键内容摘要。
  2. 更新前先验概率：这次更新前你对"该标的分红/基本面判断成立"的信念强度（0-100%）。首次记录可设为50%（中性）或参考历史信息设定。
  3. 似然比 L：新数据在"判断成立"世界里出现的概率 ÷ 在"判断不成立"世界里出现的概率。L>1代表数据支持你的判断，L<1代表数据反对，L=1代表数据不提供信息增量。这是最需要主观判断的输入，建议提前写好判定标准再看数据，避免事后合理化。
  4. 后验概率：自动计算，公式为 后验 = (L×先验) / (L×先验 + (1-先验))
  5. 预期上行% / 预期下行%：情景判断下的收益/亏损空间估计（如PB修复空间 vs 继续下跌空间）。
  6. 赔率b、全凯利f*、1/2凯利、1/4凯利：自动计算。f* = (b×p - q) / b，其中 p=后验概率，q=1-p。
为什么要打折用分数凯利，而不是全凯利

全凯利假设 p 和 b 是精确已知的概率参数，但实际上这些都是你的主观估计，存在误差。凯利公式对估计误差高度敏感——参数稍微高估，全仓位建议就可能大幅偏激进。分数凯利（1/4～1/2）用较低的长期增长率换取大幅降低的破产/深度回撤风险，是实务中更稳健的做法，尤其适合单一标的、样本量有限的低频决策场景。

重要提醒（务必阅读）
  1. 本工作表是决策辅助计算器，不构成投资建议。所有先验概率、似然比、预期收益/亏损都是主观估计，计算结果的可靠性完全取决于输入判断的质量。
  2. 凯利公式假设可重复的独立押注；但你对单一标的的一次性判断本质上是小样本事件，公式结果应视为仓位"参考上限"而非精确解。
  3. 多标的组合仓位不能简单相加。周大生、泸州老窖、五粮液同属消费/白酒链条，平安银行与大盘系统性风险相关，在系统性下跌时会同涨同跌，并非真正独立。建议在单标的凯利仓位基础上，再叠加一层组合集中度折扣。
  4. 贝叶斯更新最怕先验锚定和确认偏误：建议在看到新财报前，先把先验概率和似然比判定标准写下来，再对照数据打分，而不是看完数据后回头找理由。

补充说明 — 三大核心组件角色定位

在量化投资中，这三个工具分别扮演了资产库、指挥官和兵力分配器的角色：

所有单元格均可直接点击编辑
核心组件 扮演角色 核心解决问题 具体因子
高股息策略 底层资产库 (Alpha源) 解决"买什么"的问题。过滤掉高估值、高投机资产，锁定具有稳定现金流和高安全边际的标的。 静态股息率，动态股息率；股利支付率，现金率健康度（经营现金流净额/净利润、自由现金流率等），盈利成长性，高分红动机（大股东持股集中度、历史分红惯例）
贝叶斯定理 动态概率引擎 解决"胜率多大"的问题。随着季报、宏观数据的披露，动态更新标的"维持高股息"或"股价上涨"的后验概率。 —
凯利公式 资金分配器 (Risk Control) 解决"买多少"的问题。基于贝叶斯输出的胜率和历史波动的赔率，计算出长期对数收益率最大化的最优仓位。 —

☁️ 云端同步设置

输入 GitHub Personal Access Token (PAT) 以启用跨设备数据同步。Token 仅存储在当前浏览器本地，不会上传到任何服务器。

获取 Token 步骤：
  1. 打开 GitHub Settings → Personal access tokens → Fine-grained
  2. 点击 Generate new token
  3. Token name 随意填（如 portfolio-sync），Expiration 选 No expiration 或一年
  4. Repository access → 选 Only select repositories → 选 High-Dividend-Strategy
  5. Permissions → Repository permissions → Contents → 选 Read and write
  6. 点击 Generate token，复制生成的 token 粘贴到下方
